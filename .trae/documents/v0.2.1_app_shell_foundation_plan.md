# Plan: v0.2.1 - App Shell Foundation

This plan outlines the exact steps to implement the App Shell Foundation as defined in the `v0.2.1` build sheet. The process is broken down into a sequence of discrete, verifiable commits to ensure a clean and disciplined build.

## 1. Branch Creation

- Create a new Git branch: `feature/v0.2.1-app-shell`.

## 2. Commit 1: `feat(nav): add typed application navigation config`

- **Create File:** `apps/frontend/src/types/navigation.ts` with the specified `NavItem` type.
- **Create File:** `apps/frontend/src/config/navigation.ts` with the centralized `APP_NAV` configuration array.
- **Action:** Stage and commit these two files with the exact commit message.

## 3. Commit 2: `feat(shell): add shared authenticated app shell`

- **Create File:** `apps/frontend/src/components/app-shell/app-shell.tsx` with the pure structural wrapper component.
- **Action:** Stage and commit this single file.

## 4. Commit 3: `feat(shell): add app sidebar and primary navigation`

- **Create File:** `apps/frontend/src/components/app-shell/nav-main.tsx` to render the primary navigation links.
- **Create File:** `apps/frontend/src/components/app-shell/app-sidebar.tsx` to render the sidebar, including the brand block and the role-filtered main navigation.
- **Action:** Stage and commit these two files.

## 5. Commit 4: `feat(shell): add app header and user menu`

- **Create File:** `apps/frontend/src/components/app-shell/nav-user.tsx` for the user menu block.
- **Create File:** `apps/frontend/src/components/app-shell/app-header.tsx` to provide the shared page header.
- **Action:** Stage and commit these two files.

## 6. Commit 5: `refactor(app): render authenticated routes inside shared shell`

- **Create File:** `apps/frontend/src/lib/utils/get-page-title.ts` for the optional page title utility.
- **Create File:** `apps/frontend/src/components/app-shell/app-shell-client.tsx` to bridge server session data with the client-side `usePathname` hook.
- **Update File:** Modify `apps/frontend/src/app/(app)/layout.tsx` to use the new `AppShellClient` and pass the user session data down.
- **Action:** Stage and commit these three files.

## 7. Integration Checkpoint

- **Verify Stability:** After Commit 5 is complete, perform a runtime verification.
  - **Git Status:** Run `git status` to ensure the working tree is clean.
  - **Run Dev Server:** Execute `pnpm --filter frontend dev`.
  - **Manual Verification:** Manually click through the application to verify:
    - `/dashboard`, `/organizations`, and `/opportunities` all load correctly inside the new shell.
    - The sidebar and header persist across these route changes.

- **Rollback Rule:** If this checkpoint reveals any route breakage, auth regressions, or client/server rendering errors, **stop**. The issues must be fixed within the scope of Commit 5 before proceeding.

## 8. Commit 6: `refactor(ui): standardize authenticated page containers`

- **Scope Limitation:** This step is strictly limited to cleanup. Only remove elements that are clearly duplicating the shared shell’s global header or wrapper structure.
- **DO NOT:** Do not redesign content areas, change data fetching, or alter page business logic.
- **Action:** Inspect and trim redundant wrappers/titles from pages within the `apps/frontend/src/app/(app)/` directory.
- **Action:** Stage and commit any modified page files.

## 9. Final Verification

- **Git Status:** Run `git status` to confirm the working tree is clean.
- **Run Dev Server:** Execute `pnpm --filter frontend dev`.
- **Manual Acceptance:** Manually perform the full acceptance checklist provided in the build sheet to verify all functionality.
