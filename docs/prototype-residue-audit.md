# Prototype Residue Audit — TUF Ops Frontend (`apps/web/src/`)

**Date:** 2026-07-16
**Scope:** `apps/web/src/` source files only (`.ts`/`.tsx`; compiled `.js`/`.d.ts`/`.map` build artifacts checked into `src/` were excluded from classification but are themselves residue — see Finding R-18).
**Patterns searched:** `mock`, `demo`, `fixture`, `seed`, `DATA_MODE`, `VITE_DATA_MODE`, `fallback`, `temporary`, `hardcoded`, `fake`, `sample`, `sessionStorage`.
**Rule:** Audit only. No application code was changed.

---

## Executive summary

| Severity | Count | Headline |
|---|---|---|
| CRITICAL | 3 | Reports page renders a static all-zero summary from `mockSalesData`; Territory pages/map render stats computed from one hardcoded test org + hardcoded team members; ecosystem referrals (business records) live only in `localStorage`. |
| SUSPECT | 8 | Seeded executive badges on every ADMIN login; client-side certification fallback in Locker Room Simulator; hardcoded fallback user id `u-rep-jason-mulder`; academy certification state (coach reviews / mission statements / acknowledgments) in `localStorage`; fabricated 65/35 paid/pending commission split labeled "(mock)" in prod UI; ADMIN client-side role switcher; dead Earnings components computing from mock arrays; dev-only training enrollment localStorage fallback. |
| HARMLESS | ~12 groups | Type-only imports from `mockSalesData`, no-op stubs, boot-time localStorage cleanup, sessionStorage toast queue, sales-domain words ("mockup", "demonstrate", "sample kit") that match patterns but are curriculum copy, UI/document fallbacks, test mocks. |

**Good news:** No `DATA_MODE` / `VITE_DATA_MODE` switches remain anywhere in `apps/web/src/`, `vite.config.*`, or web `.env*` files. Auth (`authenticateWithPin`, `fetchCurrentUser`) is fully API-backed; identity does **not** come from a hardcoded source. `getActiveUserByRole` reads the API-backed user cache, not hardcoded users. `v085DataCleanup` is still wired into boot but only deletes/archives legacy keys (its `seedBaselineUsers()` call is a no-op).

## Answers to the specific questions

1. **`data/mockSalesData.ts`** — Still exports RUNTIME data, not just types: `teamMembers` (4 hardcoded users), `organizations` (1 baseline smoke-test org), `opportunities`/`orders`/`activities` (empty arrays), `reportsSummary` (static zeros), `opsWorkspaceQueue` (empty). Runtime consumers: `reportsService.ts` (**rendered on ReportsPage — CRITICAL**), `territoryMock.ts` (→ territory pages — CRITICAL), and three **dead** Earnings components (SUSPECT). All other imports are type-only.
2. **`data/territoryMock.ts`** — Exports runtime `territories`, `repCoverage`, `untouchedAccountsQueue` computed from the mock arrays above. Imported at runtime by `territoryService.ts`, which feeds `useTerritories` → TerritoryPage, TerritoryMapPage, TerritoryCommandMap. **CRITICAL.**
3. **`getActiveUserByRole`** — Not hardcoded: it filters `syncUserCache`, which is populated from `GET /users`. The risk in the `auth.ts updateRole` flow is client-side impersonation UX + an empty-cache race, not fabricated identity (R-6, SUSPECT).
4. **VITE_DATA_MODE** — No switches remain anywhere in the web app (R-20).
5. **`seedExecutiveProfile`** — Seeds 15 fabricated gold/black achievement badges into the localStorage qualifications store on every **ADMIN** login (`auth.ts:85`). Cosmetic/gamification, but applies to all admins, not just the founder (R-5, SUSPECT).
6. **`v085DataCleanup.ts`** — Yes, still imported: `main.tsx:10` and executed at `main.tsx:24` on every boot. Behavior is purely destructive/archival of legacy keys; `seedBaselineUsers()` is a no-op. Safe to keep short-term, scheduled for deletion (R-12, HARMLESS).

## Recommended remediation order

1. **R-3 / R-4 (CRITICAL):** Replace `reportsSummary` and `territoryMock` aggregates with API-derived data; delete `territoryMock.ts` and the runtime exports of `mockSalesData.ts`.
2. **R-13 (CRITICAL):** Persist ecosystem referrals server-side.
3. **R-11 / R-9 / R-14:** Kill the offline certification pass, the `u-rep-jason-mulder` fallback, and localStorage certification artifacts.
4. **R-5 / R-6 / R-8:** Gate or remove executive badge seeding; replace client-side role switching; fix the "(mock)" commission split.
5. **R-7 / R-16 / R-17 / R-12 / R-18:** Delete dead Earnings components, retarget type imports to `@tuf/shared`, remove stubs, retire `v085DataCleanup`, and purge compiled artifacts from `src/`.
